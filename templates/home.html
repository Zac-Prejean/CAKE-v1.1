How to get started with ShipStation and the AutoDesigner

  1. Access ShipStation using your personal login credentials.
  2. Choose the orders you wish to design, prioritizing them based on order date or priority level.
  3. Navigate to the "Other Actions" tab located at the top of the page, and find "Export Orders".
  4. Select the "Export Line Items" option, followed by "Item Skus W Options."
  5. Click on the "Export" button to complete the process.
  6. Proceed to the Precheck.
    7. Go to ShipStation

What to look for in Precheck

ALL ORDERS MUST GO THROUGH THE PRECHECK! The PreCheck page handles CSV files containing product information and personalization data. It allows users to upload a CSV file, preview and edit the personalization fields, validate SKUs, and download an updated CSV file with the changes.

  1. Locate and open the CSV file obtained from ShipStation
  2. Review each order meticulously, checking for any inconsistencies or anomalies.
  3. Take note of the different markings next to the order SKU, which indicate the following:
    • Green mark - The order is recognized and ready for designing.
    • Red mark - The order is unrecognized and cannot be designed.
    • Yellow mark - The order has a varying number of personalization lines, requiring close attention during the design process.
    • Red mark with a bug - The order has an unusual setup in the CSV file, which may lead to incorrect designing. Proceed with caution.
  4. Utilize the "Cancel" button to remove any items with a red mark from the CSV file. Please note that this action will only remove the individual item, not the entire order.

    5. By following these steps, you can efficiently manage and design orders based on their corresponding status indicators.

    Go to the Precheck Page

Designing and after-check with the Design page

The Designer page processes a CSV file containing product information and generates personalized images for each product based on the user's input. The images are saved to the user's 'Downloads' folder.

  1. Locate the "updated_csv" file in the Downloads folder, which was generated in the previous step, by pressing the browse button.
  2. Upload and submit the CSV file. Select the necessary checkmarks if Labels or a Pick List are required. Monitor the progress window at the bottom of the screen, which will confirm when the design process is complete.
  3. Access the newly created designs in the Downloads folder. These designs will be named with the current date, time, and an index number.
  4. Thoroughly review each design to ensure its accuracy. This step serves as the final quality control checkpoint before proceeding to the printing stage. Exercise caution and attention to detail.
  5. In some departments, specific symbols are used during the shipping process to facilitate smoother operations. These symbols include:
    • Rabbit: This symbol indicates that the order is either expedited or a redo, requiring prompt attention and faster processing.
    • Shipping box: This symbol signifies that the order contains more than one item. If possible, these items should be kept together to maintain efficiency.
    • Multiple cubes: This symbol represents that the order consists of multiple instances of the same design on different items. It is crucial to ensure consistency across the various items.
    • Multiple tags: This symbol denotes that the order includes multiple SKUs. Pay close attention to the different product types and their respective design requirements when processing these orders.

    6. By following these steps, you can efficiently manage and verify the design process, ensuring that all designs are accurate before moving on to the printing phase.

    Go to the Design Page

When to use the TemplateMerger

The Template Merger Page is for processing and combining multiple images loaded by the user. It also provides a preview for the final combined image.When processing large batches of prints simultaneously, follow these steps using the TemplateMerger:

  1. Select the appropriate item format from the format bar, ensuring it matches the products to be printed on.
  2. Browse the Downloads folder to locate the accurate designs that were created and verified in the previous steps.
  3. Click "Export" to initiate the batch printing process. Please note that these files can be quite large, and it may take some time for the browser to generate them.
  4. If you encounter an issue where the downloaded file size is "0 bytes" and appears blank when viewed as a .png, it might be a browser-specific problem. To resolve this issue:
    • Close your current browser.
    • Open a different web browser (e.g., Google Chrome, Mozilla Firefox, Microsoft Edge, or Safari).
    • Repeat the previous steps for creating and exporting large batches of prints using the TemplateMerger in the new browser.

    By following this procedure, you can efficiently create large batches of prints while maintaining consistency and quality throughout the process.

    Go to the TemplateMerger Page

Item Display and Status Update Process

This process fetches item details, images, and status updates from a server and displays the information on a web page. The "Scan-In" page is the first step of this process. With the Scan-In the user accepts the picked items and prepares them to be printed on.

  1. Scan the Item-ID QR code on the given label.
  2. An image will appear on screen with a indicator to the side on which the item needs to be printed on.
  3. The Scan-In Station states that users are excepting the Item's label and will start the creation process.
  4. The Print Station enables users to view a final sample image for evaluating the item's placement before proceeding. It also states that the Item has been created and is ready for processing to the next step.
  5. The numbers must be scanned using the AutoHotKeys tab. To ensure proper functionality, this station should be opened with Google Chrome and have access to the Everything Folder.
  6. Lastly, there is the Scan-Out Station, which serves as the final quality control checkpoint for most orders. Users must ensure that all information in the item image and details is accurate before proceeding further.
  7. A confirmation notification will appear for each scanned number. Once confirmed, the shipping label will automatically print for the item, unless the order contains multiple items. In that case, the CubbyHole system is utilized to manage multi-item orders.
Go to the Scan-In Station

What to do with Orders that are Not Quite Right

  1. The Redo Station is designed to monitor and track all orders that contain errors or discrepancies.
  2. Scan the Order Number and Item ID associated with the item that needs to be returned for correction.
  3. Select the appropriate reason for the item's error from the provided list of reasons.
  4. Valid reasons for which an item may be incorrect include:
    • WRONG ITEM
      The item received is not the one that was ordered.
    • WRONG SIZE
      The size of the item received does not match the size ordered.
    • WRONG COLOR
      The color of the item received is different from the color ordered.
    • MISSING ITEM
      An item that was supposed to be included in the order is missing.
    • Bad Plating
      The plating on the item is of poor quality or defective.
    • Bad Sanding
      The sanding on the item is inadequate or uneven.
    • Operator Error
      A mistake was made by the operator during the manufacturing process.
    • QC Fail
      The item did not pass quality control checks.
  5. An "ERROR" status will be given to the Item.
Go to the Redo Page

Using the CubbyHole System for Storing Multiple Items

  1. The CubbyHole is intended to store all orders containing multiple items, facilitating easier searchability and retrieval.
  2. The Bins may display a variety of colors. These colors include:
    • Blue-Bin is still waiting on items and not yet complete.
    • Green-Bin has been completed and is ready to be shipped.
    • Pink-Bin has just been added to the Cubby page within the last 10 minutes.
    • Yellow-Bin has been created over 24 hour ago. The missing items should be searched in order to complete the bin.
    • The Red Bin was created over 48 hours ago and is likely overdue.
  3. Utilize the search bar to locate Order Numbers or Item IDs.
  4. Select a bin or use the search bar to display the contents of a bin.
  5. If the bin contains all items required to complete the order, its color will change to green. The order can then be completed by clicking the 'Complete' button.
  6. If a bin needs to be closed because the order has already shipped or the bin is no longer needed, the 'Delete' button can be used to close the bin.
Go to the Cubby Page

Shipping Out and Admin Privileges

  1. The Ship-Out Station is used to efficiently ship orders without the need for third-party sites.
  2. Scan the order number to initiate the shipping process. Ensure that the information matches the order details.
  3. Most pages within the CAKE system require a sign-in, and some are restricted to Admin use only. This ensures that orders and items are processed accurately and sent to the correct locations.
  4. Only cards with the numbers 01 or 02 have access to Admin privileges.
Go to the Shipping Station

Keeping Score with the TallyPage

  1. The Tally Page is designed to track and record a user's input and output.
  2. The numerical data on the page represent the quantity and type of work the user has accomplished for the day.
  3. A gold circle, representing the user with the highest score, is positioned at the top of the table. This is followed by silver and bronze circles for the subsequent top performers.
  4. Two numbered circles are located on the left-hand side of the page. The upper circle indicates the number of items that have been added to CAKE for the day, while the lower circle shows the total number of items completed for the day so far.
Go to the Tally Page

Warnings and Confirmation

  1. CAKE has been streamlined to ensure maximum efficiency. Robust systems are in place to guarantee that orders and items are correctly positioned and on schedule for accurate shipping.
  2. These systems include:
  3. MISMATCH STAGE: This stage indicates that the scanned item is either not in the correct stage for progression or cannot be located within the system.
  4. If the user is redirected to the homepage when attempting to sign in, it indicates that either they are not registered in the system or the system requires a restart. When a PC is powered off or put into sleep mode overnight, the system will lock out users as a safety feature, necessitating a restart.
  5. If the user is redirected to the homepage when attempting to sign into an ADMIN restricted page, it usually indicates that the user does not have the necessary access permissions.
  6. If the user is attempting to sign and the page goes blank, it usually indicates that the user input the wrong sign-in code.
  7. If the user misses the given cubby number, they can either locate it by scanning the Order Number in the CubbyHole search bar or ask an Admin to find it on the ADMIN Page. If no cubby number is listed on the ADMIN Page, it might indicate that the user pressed the SUBMIT button before a number was assigned. The SUBMIT button on the Sign-Out Page is intended only for items that need to be manually input. The system is designed to automatically assign cubby numbers.
